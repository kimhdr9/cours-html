Les premières commandes Docker

Sur le site hub.docker.com, il existe un grand nombre d'applications : comme des serveurs web, apache, nginx, des bases de données, postgresql, mongodb, mysql, des distributions linux, ubuntu, alpine, des langages python, javascript, des applications nextcloud..

Charger une image

Récupérer une application, par exemple la distribution linux alpine dans le même principe que git :
docker pull alpine
Cela renvoi un message d'erreur, car il a besoin de connaitre la version à récupérer,
docker pull alpine:latest
Dans le vocable de docker, nous venons de prendre une image de "alpine".

Les commandes sur les images

Pour afficher les images chargées :
docker images ls
On peut remarquer la synthaxe de docker s'inspire des commandes unix comme ls, rm. On peut tou simplement taper la commande suivante :
docker images
Noter l'image id tout comme git c'est une séquence hexedécimale assez longue, ce qui est affiché est une partie tronquée.
